## ADDED Requirements

### Requirement: Image Generator mode reuses the existing generate-image pipeline
The system SHALL implement `image-generator` as a dedicated chat mode on top of the existing `generate-image` Edge Function, rather than introducing a separate Gemini image backend.

#### Scenario: Send prompt in Image Generator mode
- **WHEN** user sends a prompt with `image-generator` mode selected
- **THEN** the frontend SHALL call the existing `generate-image` Edge Function
- **AND** SHALL pass the configured Gemini image system model as the effective model
- **AND** SHALL NOT call the `agent` Edge Function for that request

### Requirement: Gemini image generation uses the existing shared provider path
The image generator mode SHALL use the existing Gemini image provider, upload, asset creation, and points deduction path already used by `generate-image`.

#### Scenario: Successful generation and upload
- **WHEN** Gemini generates an image successfully
- **THEN** the system SHALL upload the image to the project's asset storage
- **AND** SHALL create the corresponding asset record
- **AND** SHALL continue using the existing `jobs` output contract consumed by the frontend

#### Scenario: Generation failure
- **WHEN** Gemini generation fails
- **THEN** the system SHALL surface the failure through the existing `jobs` / error handling path
- **AND** the frontend SHALL show the failure using the current generation error behavior

### Requirement: Aspect ratio and resolution support
The Gemini image generator SHALL support aspect ratio and resolution configuration passed from the frontend. The supported values SHALL match the existing `AspectRatioSelector` and `ResolutionSelector` options.

#### Scenario: Custom aspect ratio and resolution
- **WHEN** a generation request includes `aspectRatio: '16:9'` and `resolution: '2K'`
- **THEN** the request SHALL be forwarded through the existing `generate-image` contract
- **AND** Gemini generation SHALL use those values subject to membership and model limits

#### Scenario: Default values
- **WHEN** a generation request omits `aspectRatio` or `resolution`
- **THEN** the system SHALL use the same defaults as the current image generation flow

### Requirement: Placeholder, job subscription, and canvas placement are preserved
Image Generator mode SHALL preserve the existing placeholder, job subscription, Realtime, and canvas placement flow already implemented by `useGeneration`.

#### Scenario: Placeholder during generation
- **WHEN** an image-generator request is in progress
- **THEN** the system SHALL display a placeholder on the canvas
- **AND** SHALL remove or replace it using the current job completion behavior

#### Scenario: Place generated image on canvas
- **WHEN** the job completes successfully
- **THEN** the frontend SHALL reuse the existing `addImage` op execution and persistence logic
- **AND** SHALL preserve current smart placement, local-op deduplication, and `saveOp` behavior

### Requirement: Reference image via @mention
The Gemini image generator mode SHALL support receiving a reference image through the existing `@mention` asset reference mechanism in ChatInput.

#### Scenario: User references a project asset with @mention
- **WHEN** user selects a project asset via `@mention` before sending a message in Image Generator mode
- **THEN** the frontend SHALL include that asset URL in the existing image generation request
- **AND** the backend SHALL fetch and pass the image to Gemini using the current reference-image handling logic

#### Scenario: Reference image ownership validation
- **WHEN** a request includes a reference image URL
- **THEN** the backend SHALL only accept URLs that belong to the current project assets or trusted storage origins
- **AND** SHALL reject untrusted external URLs

### Requirement: Assistant message format for Image Generator mode
Messages generated by Image Generator mode SHALL continue to be stored in the existing `messages` table with `role: 'assistant'`.

#### Scenario: Image Generator message
- **WHEN** an Image Generator mode request completes successfully
- **THEN** the assistant message SHALL include the generated image in the same metadata fields already used by current image generation
- **AND** `metadata.mode` SHALL be set to `'image-generator'`
- **AND** `metadata.modelName` SHALL identify the effective Gemini image model used
