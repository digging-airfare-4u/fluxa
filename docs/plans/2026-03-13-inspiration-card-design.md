# 灵感发现卡片轻量化设计（小红书风图文平衡流）

## 背景

用户希望优化首页与发现页的灵感卡片体验，明确提出两个问题：

1. 当前卡片偏大，视觉重量过重。
2. 当前瀑布流虽然使用了 columns 布局，但整体仍然显得过于整齐，缺少类似小红书那种“自然参差、顺手想继续刷”的内容流体验。

用户希望的新方向不是纯图片墙，也不是杂志式精致卡片，而是 **图文平衡型的小红书风瀑布流**：
- 图片先抓人
- 保留简短标题帮助判断内容
- 信息区更轻
- 不希望所有卡片都机械对齐

## 当前实现观察

### 首页灵感区
- 文件：`src/app/app/page.tsx`
- 当前使用 `columns-2 sm:columns-3` 瀑布流展示灵感卡片。
- 区域定位正确，但卡片仍偏“作品展示卡”，预览流感不够强。

### Discover 页
- 文件：`src/app/app/discover/page.tsx`
- 当前使用 `columns-2 sm:columns-3 md:columns-4 xl:columns-5` 瀑布流。
- 顶部搜索、排序、分类功能完整，但卡片本体仍偏重，整体更像规整作品列表而不是内容流。

### 卡片组件
- 文件：`src/components/discover/PublicationCard.tsx`
- 当前卡片特征：
  - 完整白底盒子
  - 弱边框 + 阴影
  - 图片 + 标题 + 作者 + 浏览量 + 显性点赞/收藏按钮
- 问题在于：
  - “卡片壳”明显，偏组件化/商品化
  - 操作按钮抢视觉，内容感不足
  - 每张卡信息结构过于统一，导致瀑布流虽然错落但整体观感仍整齐

## 设计目标

本次调整目标不是重做发现系统，而是把灵感流的气质从“规整作品卡陈列”调整为“轻量内容笔记流”。

核心目标：
- 弱化卡片外壳感
- 提高图片视觉权重
- 保留必要标题与作者信息
- 减少功能按钮存在感
- 通过图片高度差与文本占用差制造自然参差感
- 保持首页与 Discover 页风格统一，但定位有所区分

## 方案对比

### 方案 A：保守缩小版
- 只缩小现有卡片尺寸
- 保留现有信息结构与操作区

**优点**：改动小

**缺点**：只能得到“小一点的旧卡片”，无法真正形成小红书风内容流感

### 方案 B：图文平衡型轻量卡流（推荐）
- 弱化卡片盒子感
- 图片作为主视觉
- 标题保留 1–2 行
- 底部保留一行轻 meta（头像 + 用户名 + 一个轻互动指标）
- 通过不同图片高度、标题占用差形成自然错落

**优点**：兼顾浏览效率、内容判断与灵感流氛围，最适合当前产品阶段

### 方案 C：极致图片流
- 几乎只展示图片
- 标题和 meta 极度弱化

**优点**：更强的“逛图”感

**缺点**：内容辨识度下降，不适合依赖标题帮助理解的作品内容

**结论**：采用方案 B。

## 最终设计方向（已确认）

### 1) 卡片定位
卡片应从“作品展示卡”转向“内容笔记单元”。

目标感受：
- 用户第一眼先看图
- 再通过短标题快速判断要不要点开
- 不会感到每张卡都被完整包装展示
- 页面整体更像连续内容流，而不是组件化作品墙

### 2) 卡片结构
建议卡片收敛为：
- 图片：主视觉区域，占据绝大部分注意力
- 标题：1–2 行，作为快速判断信息
- 底部轻 meta：头像 + 用户名 + 一个轻互动指标

不再保留重交互区作为默认状态。

### 3) 容器风格
应弱化以下元素：
- 明显白底盒子
- 明显边框
- 明显阴影

保留：
- 柔和圆角
- 适度留白
- 内容本身的呼吸感

设计原则：
- 不是做粗糙内容墙
- 而是做轻、松、自然的灵感流

### 4) 图片策略
图片需要成为绝对主角，并承担主要节奏变化来源。

原则：
- 宽度统一
- 高度变化
- 不强行裁成统一比例
- 通过高低错落建立瀑布流节奏

目标不是“乱”，而是“有秩序的不齐”。

### 5) 标题策略
标题保留，但明显收轻：
- 1–2 行
- 用于快速判断内容
- 不承载完整说明功能
- 不增加副标题或长描述

### 6) Meta 策略
建议底部只保留一条轻信息：
- 左边：头像 + 用户名
- 右边：一个轻互动指标（优先考虑点赞）

浏览量不建议作为首层核心信息展示，因为它会增加工具化/数据卡气质。

### 7) 操作区策略
当前点赞/收藏显性按钮会显著提升卡片“功能组件感”。

建议：
- 首页灵感区默认不显式展示操作按钮
- Discover 页也尽量弱化默认操作呈现
- 先让用户浏览内容，再进入操作

## 参差感设计原则

卡片不应通过完全不同的样式制造“乱”，而应通过统一规则下的轻变化制造“松”。

推荐变化来源：
1. 图片高度不同
2. 标题占行数不同（1 行 / 2 行）
3. meta 视觉占用轻微变化

建议维持：
- 宽度统一
- 间距统一
- 高度变化
- 文本占用变化

这样可以维持高级感与可浏览性。

## 页面分工

### 首页灵感区
- 文件：`src/app/app/page.tsx`
- 定位：**Preview feed**

设计要求：
- 更轻
- 更少信息
- 更像“顺手逛到几张好内容”
- 主要任务是吸引用户点击进入 Discover 页

建议：
- 手机 2 列，桌面 3 列
- 展示 6–8 张即可
- 保留图、短标题、轻 meta
- 默认不展示显性交互按钮
- `See all` 应作为明显入口承担导流功能

### Discover 页
- 文件：`src/app/app/discover/page.tsx`
- 定位：**Full browsing feed**

设计要求：
- 与首页同一视觉语言
- 信息略完整一些
- 支撑持续浏览与无限滚动
- 顶部筛选工具保持可用，但不抢内容主角地位

建议：
- 手机 2 列，平板 3 列，桌面 4 列，超大屏可到 5 列
- 卡片仍然轻量，但允许比首页略完整
- 保持瀑布流节奏，不回退到规则 grid

## 建议删除或弱化的元素

### 高优先级弱化项
1. 卡片完整白底盒子感
2. 明显边框和阴影
3. 默认显性 Like / Bookmark 按钮
4. 浏览量作为首层数据
5. 每张卡完全一致的信息区结构

## 建议保留的元素

1. 瀑布流布局本身
2. 标题（1–2 行）
3. 作者信息（头像 + 名字）
4. 圆角与适度留白
5. 首页与 Discover 页之间统一的视觉语言

## 最小落地顺序

### 第一步：先调整卡片组件
- 文件：`src/components/discover/PublicationCard.tsx`
- 去盒子感
- 压缩信息区
- 弱化/去除显性操作按钮
- 保留图 + 标题 + 轻 meta

### 第二步：再调整首页与 Discover 页的瀑布流节奏
- 文件：`src/app/app/page.tsx`
- 文件：`src/app/app/discover/page.tsx`
- 首页更轻、更密、更像预览
- Discover 更稳、更完整，但保持同风格

### 第三步：最后微调 Discover 顶部区域与 skeleton
- 让浏览工具后退
- 让 loading 状态也符合轻量卡流的视觉语言

## 范围边界（YAGNI）

本次不做：
- 发现页数据查询链路重构
- 新推荐算法
- 大规模视觉品牌重设计
- 完全不同类型的卡片模板系统
- 宽窄不同的拼贴式布局

本次只聚焦：
- 卡片气质
- 信息层级
- 瀑布流节奏
- 首页与 Discover 的内容流一致性

## 设计结论

最终方向为：

**把灵感发现卡片从“规整的作品展示卡”改成“轻量的内容笔记流”：保留瀑布流，弱化卡片壳，突出图片，保留短标题和轻作者信息，去掉重操作区，通过图片高度和信息占用差形成自然参差。**
